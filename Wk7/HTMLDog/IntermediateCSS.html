CSS can also be grouped (separation using ',') and nesting (use space to separate the following subordinate elements). link or unvisited link should be blue, while visted link should be purple.

CSS rule conflicts: more specific = greater precedence. If the selectors are the same, then the last one wins.

Calculating specificity: ID Selector (#) = 100, Class Selector(.) = 10, HTML element selector(such as 'p') = 1; then add them together. The bigger number wins.

    For navigation bar, using "{display: inline}" would make all list items appear next to each other, instead of line by line display of list items. There are also {display:block} to make keep each element as a clickable block. Also 'inline-block' to have them inline and at same time with a clickable box.

Also {display: none;} and {visibility: hidden;} are different, where first one get rid of the box completely, the second one is just hide the box for display, but keep the invisible box and its flow in place.

    Tables not only have 'tr' and 'td' for table-row and table-cell property values in HTML Tables; but also has CSS way of using 'display' property(But be aware of older browsers or user turn off CSS could cause serious trouble. So it is recommended to use HTML Table instead for now), such as below:

  • table-column
  • table-row-group
  • table-column-group
  • table-header-group
  • table-footer-group
  • table-caption
  • inline-table: it will set the table without line breaks before and after it.

    • 'run-in' makes a box either in-line or block, depending on the display of its parent.

Pseudo Elements: four of them, using the form of selector:pseudoelement {property: value;}

  • First Letter
  • First Line
  • 'before' pseudo element uses in conjunction with the 'content' property, without touch the HTML. (not recommended).
  • 'after' with 'content'. Same as above.

    • BTW, CSS3 specs suggest pseudo elements should include two colons, such as p::first-line instead of p:first-line.

Page Layout using 'position' property to define a box as 'static', 'relative', 'fixed' or 'sbsolute'; 'float' to 'left' or 'right'; (and 'clear' of 'left', 'right' or 'both' to cancel the 'float'). 'clear: both' is esp important to 'footer'.

Remember: they can apply to boxes within those boxes too. So the combination of positioning, floating, margins, padding and borders, would be enough to represent any web design required.
