Universal, child, adjacent selectors

Universal Selector is just a '*' or asterisk, to target everything in the content (after the condition of any element before it). Usually used to cancel out default styles of the browser.

Below is the usage of child selectors: only 'Cats' &. 'Apes' are red boxed, not those grandchildren elements like 'Felis' or 'Pan'.

  • Cats
    • Panthera
    • Felis
    • Neofelis
  • Apes
    • Pongo
    • Pan
    • Homo

Adjacent Selector use a plus sign like '+'; like following example:

Clouded leopards are cats that belong to the genus Neofelis.

There are two extant species: Neofelis nebulosa and Neofelis diardi.

So only the first paragraph followed 'h4' title are bold. But, if you replace '+' with '~' in CSS statements, then all the following paragraphs under that title will be affected.

Advanced CSS3 Colors: Hue, Saturation, Lightness (HSL), and transparency.

The last value of '0.8' in rgba, is 80% of opaque or 20% of transparent. So it is 80% white here.

RGB as web color is always Red, Green and Blue. Here is what it means for 'hsl(36,100%, 50%):

  • '36' is Hue value, that angle from 0 to 360 degree of a typical color wheel, are:
    • 0 and 360 is red
    • 120 is green
    • 240 is blue
    • So 36 is between red and green here.
  • 100% is Saturation of hsl, that 0% is none and 100% is the whole in the level of grey. Here 100% is the full whack.
  • 50% is the Lightness of hsl, that 0% is black and 100% is white. 50% is 'normal'.
  • So this example of hsl(36,100%, 50%) is: orange, 100% rich of colour, and 50% lightness or normal vibrant. It is the equivalent of #ff9900, #f90, or rgb(255,153,0).

HSLa (a for alpha transparency, just like RGBa), is the funky way of combine transparency and HSL, which is supported by IE9 and above.

At-Rules: @import, @media, @font-face

Importing: @import url(morestyle.css); used to bolt another stylesheet onto this existing one. It is used to break down a lot and complex stylesheet into smaller files for easy management. It must be placed at the top of a stylesheet, before any other rules.

Targeting media types: @media can be used to apply styles to a specific media, such as print. 


                @media print {
                    body {
                        font-size: 10pt;
                        font-family: times, serif;
                    }

                    @navigation {
                        display: none;
                    }
                }

Values of @media can include:

  • screen
  • print
  • projection
  • handheld
  • all (for all above)
  • or a comma-separated list of more than one of above, such as @media screen, projection { /* .. */}
  • And also: media query, which is the last section of this article.

Embedding Fonts


        @font-face {
            font-family: "font of all knowledge";
            src: url(fontofallknowledge.woff), local(fontofallknowledge), local("font of all knowledge");
            font-weight: 400;
            font-style: normal;
        }

        p {font-family: "font of all knowledge", arial, sans-serif;}

If the font has been downloaded and existed in local directory, then you can use 'local()' as above. Otherwise just omit them.

This enabled you to use other fonts outside of 'web safe' fonts such as Arial or Verdana. If you have the copyright to use it, or use Google Web Fonts (more limited selection but free).

Attribute Selectors

There is no current style here for reference, and it is 2:30AM so I won't make few now. Just copy and paste them into here, for later reference.

  • With the attribute -- shove a dotted line underneath all abbreviation with a 'title' attribute. 
    
                abbr[title] {border-bottom: 1px dotted #ccc;}
  • With the attribute and value -- apply a width of 200px only to 'input' element spedified as text boxes. 
    
                input[type=text] {width: 200px;}
  • Could also target more than one attribute at a time -- only apply a gray border to text inputs that are disabled. 
    
                input[type=text][disabled] {border: ipx solid #ccc;}
  • CSS3 further allow to match an attribute without being exact: [attribute^=something] means match with the value of an attribute that begins with 'something'. Below is making 'a' element with external links 'href=http...' style differently, that internal link won't apply for this style: 
    
                a[href^=http] {
                    padding-right: 10px;
                    background: url(arrow.png) right no-repeat;
                }
  • [attribute$=something] means match value ends with 'something'.
  • [attribute*=something] means match value contains 'something', no matter its location at beginning, middle or end.

CSS Transitions -- animate parts of design without the likes of Javascript!

The simplest level, is like state of ':link' (such as a:link) and ':hover' (such as a:hover). They are binary animation (or jump from one state to another). The transition property, is much more fowerful.

  • transition-property: which property (or properties) will transition
  • transition-duration: how long the transition takes
  • transition-timing-function: speed constant, or if accelerates or decelerates
  • transition-delay: how long to wait until starts transition
  • Only transition-duration is required for a transition to take place. the rest defaulting to:
    • transition-property: all;
    • transition-timing-function: ease;
    • transition-delay:0;
    • So, you could simply declare 'transition: .5s'; which is enough.
