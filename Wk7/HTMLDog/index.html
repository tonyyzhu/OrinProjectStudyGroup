The most commonly used file formats for images are JPEGs, GIFs and PNGS

  • JPEGs are typically used for images such as photographs, using mathematical algorithm to compress the image and will distort the original stightly. The lower the compression, the clear the image and the higher the file size.
  • GIFs are typically used for images with solid colors, such as icons and logos. It can have no more than 256 colors, but they maintain the colors of the original image. The lower the number of colors you have in the image, the lower the file size will be. It also allow any pixel in the image to be transparent.
  • PNGs are typically used for versatile images in more complex dsigns But they are not fully supported by some older browsers. It replicates colors (like GIF), but allow 16 million colors as well as alpha transparency(that is, an area could be 50% transparent).

The web is forever getting faster and faster, and users are less and less patient. Using super-high resolution images isn't doing any favors to your & your user's bandwidth (or patience). Image compression is a great tool and you need to strike a balance between image quality and image size.

Form part is something that I didn't touch in basic HTML section. It has form, input, textarea, select, and option tags. Each of them has a lot of attributes that related to XML protocol and other languages (such as JavaScript or PHP). I should look at this link in detail to patch this hole of my knowledge.

Just 'input' tag will have many type attributes:

  • type="text": a standard textbox; default if 'type' attribute is omitted.
  • type="password": similar to above, but input will be hidden.
  • type="checkbox": could also has 'checked' attribute to follow as default value.
  • type="radio": similar to above, but only one button can be selected in a group.
  • type="submit": control the text with 'value' attribute.
  • Just like 'img' &. 'br' tags, the 'input' tag does not surround any content, so doesn't require a closing tag.
