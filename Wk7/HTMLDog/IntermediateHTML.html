Span and Div: with attributes of class and is, used for CSS selectors

'span' element is in-line and usually used for a small chunk of HTML inside a line, such as inside a paragraph.

'div' or division is block-line(like having a line-break before and after it) and used to group larger chunks of code.

HTML5 has more tags for grouping and also adding meaning at same time, for those blocks that used to have to be done in 'div' tag: 'article', 'header', 'footer', and more.

blockquote and q are used for quotations. blockquote is generally used for standalone often multi-line quotations whereas q is used for shorter, in-line quotations.

Blockquotes work very nicely with the HTML5 elements 'figure' and 'figcaption', enabling a nice, semantic way to group a quotation with a citation.

Citations: HTML5 not only has 'cite' attribute (such as in 'blockquote' tag), but also has a 'cite' tag, that can be used to define the title of a work(such as a book).

'code' is used to represent any form of computer code. Further, 'var' can be used for variables (which could be a variable in anything, not just in code - it could be a variable in an equation, for example), 'samp' for sample output, and 'kbd' (keyboard) for user input.

If you add the line givevaderachuckle = true; to the destroy_planet subroutine and then type ilovejabba into the console, the big bad green Death Star laser will etch Slug Lover! on the planet's surface.

'pre' is preformatted text and is unusual in HTML tags that it takes notice of every character in it, including the white space (whereas other elements will ignore a consecutive space or a line-break, for example). It is most commonly used for blocks of code, where spacing, such as indentations, can be relevant.


    <div id="intro">
        <h1>Some heading</h1>
        <p>Some paragraph paragraph thing thing thingy.</p>
    </div>

Meta Tags

Meta tags don't do anything except for search engine indexing. Its attributes such as 'keywords' and 'description' is often used by search engines to display a description of a web page in its search results. 'keywords' meta data was used (and abused) extensively, that nowadays Google rely on the actual content of the page itself. Below is an example: 

 
    <head>
        <title>Air conditioners? YEAH conditioners!</title>
        <meta charset="utf-8">
        <meta http-equiv="refresh" content="3"><!--not recommended for sane people-->
        <meta name="description" content="This is my really, really, REALLY exciting web page about air conditioners">
        ...

'dfn' is a definition term and is used to highlight the first use of a term. Like 'abbr', the 'title' attribute can be used to describe the term. Example below:

Bob's canine mother and equine father sat him down and carefully explained that he was an allopolypoid organism.

'bdo' can be used to reverse the direction of the text, and can be used to display languages that read right to left. The value of the required attribute 'dir' can be 'ltr'(left to right) or 'rtl'(right to left). Below is an example:

god lmth

'ins' and 'del' are used to display editorial insertions and deletions respectively, just like "Track Changes" feature in word processors. It can be used on swathes of content instead of just a few text/words.

I have decided to decrease increase the amount of free ice cream that the State will provide for its citizens.

Sectioning using a number of tags to provide a cleaner, clearer semantic structure

'article' element can be used to make up a standalone section of content. 'section' element represents a more general section and could be used to split up an article, or to represent chapters, for example. It used to be 'div' for these separations and/or styling, but these tags would provide a much richer and more meaningful document.

[An introduction]

[Content]

  • That related article
  • This related article

'header' and 'footer' provide further specialized and self-descriptive sections. In below articles, the first 'footer' is the footer of the article and the second 'footer' is the footer of the page.

Alternatively…

[An introduction]

[Content]

[End notes]

[Copyright bumf]

'aside' is more like a pull-quotes or snippets of related information in an article:

Tixall

[All about Tixall]

Tixall obelisk

[A short note about Tixall obelisk]

[Maybe a bit more about Tixall]

'nav' is used to mark up a group of navigation links.

  • Tutorials
  • Reference
  • Articles
  • About us
  • In-Page Navigation for major groupings. (copyright, author, sidebar, footer, etc.)

BTW, IE8 or earlier versions don't understand these tags, although it is much nicer than old <div id="#"> elements. In these case, HTML5Shiv can be your rescue. Or, you could stay with safe old but semantically poorer headings and divs approach.
